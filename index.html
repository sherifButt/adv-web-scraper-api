Advanced Web Scraper API

A powerful, scalable API for web scraping, navigation, and AI-driven data extraction.

Key Features

AI-Powered Configuration

Generate complex scraping configurations from just a URL and natural language prompt. Let AI handle the setup. Learn more.

Advanced Navigation

Execute multi-step browser interactions, handle logins, pagination, conditional logic, and dynamic content with a powerful navigation engine. See types.

Robust Proxy Management

Automatic proxy rotation, health checks, and support for multiple sources (file, API) to ensure reliable and undetected scraping. Proxy API.

Persistent Sessions

Maintain browser sessions across jobs, preserving cookies and localStorage to bypass logins and reduce CAPTCHAs. Session Docs.

Flexible Extraction

Extract data precisely using CSS selectors, XPath, Regular Expressions, or even custom JavaScript functions.

Scalable Architecture

Built with a robust queue system (BullMQ) to handle high-volume scraping tasks efficiently across multiple workers.

API Endpoints

Explore the core functionalities of the API:

  • POST /api/v1/ai/generate-config - Generate scraping configurations using AI.
  • POST /api/v1/navigate - Execute complex browser navigation flows.
  • POST /api/v1/scrape - Perform direct scraping tasks (worker implementation pending).
  • GET /api/v1/jobs/:id - Check the status and retrieve results of asynchronous jobs.
  • GET /api/v1/proxy/... - Manage and utilize proxies.
  • GET /api/v1/session/... - Manage persistent browser sessions.
  • POST /api/v1/captcha/... - Interact with the CAPTCHA solving system.
  • GET /health - Perform a system health check.

See the full API Reference for details.

Quick Start

Get started quickly with the API:

  1. Explore the Documentation: Familiarize yourself with the concepts and available API endpoints.
  2. Generate a Config (Example): Use the AI endpoint to create a scraping configuration. Replace `YOUR_API_URL` with the actual running server URL (e.g., `http://localhost:3000`). 
    
curl -X POST YOUR_API_URL/api/v1/ai/generate-config \
-H "Content-Type: application/json" \
-d '{
  "url": "https://news.ycombinator.com/",
  "prompt": "Extract the title and URL for each story listed on the front page.",
  "options": {
    "model": "gpt-4o-mini",
    "testConfiguration": true,
    "maxIterations": 3
  }
}'
    This will return a Job ID.
  3. Check Job Status: Poll the job status endpoint until `status` is `completed`. Replace `YOUR_JOB_ID` with the ID from the previous step. 
    
curl YOUR_API_URL/api/v1/jobs/YOUR_JOB_ID
  4. Use the Generated Config: The completed job result will contain the generated JSON configuration, ready to be used with the `/api/v1/navigate` endpoint.

For more detailed examples, see the Navigation Examples and AI API Guide.
