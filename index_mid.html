API Reference

Explore the comprehensive set of endpoints that power our scraping platform

  • POST /api/v1/ai/generate-config - Generate scraping configurations using AI NEW
  • POST /api/v1/navigate - Execute complex browser navigation flows

Quick Start Guide

Get up and running with the API in minutes

  1. Installation
    Clone the repository 
    git clone https://github.com/sherifButt/adv-web-scraper-api.git
cd adv-web-scraper-api
npm install
  2. Set up environment
    Create .env file 
    PORT=3000
REDIS_HOST=localhost
REDIS_PORT=6379
# Add other configuration as needed
  3. Start the server
    Run the development server 
    npm run dev
  4. Make your first API call

    curl -X POST http://localhost:3000/api/v1/ai/generate-config \

-d '{

  "prompt": "Extract the title and URL for each story"
}'

Frequently Asked Questions

Find answers to common questions about the Advanced Web Scraper API

What is the AI Configuration Generation feature?

This feature allows you to generate a scraping configuration (`config.json`) simply by providing a target URL and a natural language prompt describing what data you want to extract. The API uses a Large Language Model (LLM) to create the configuration, potentially tests it, and can even attempt to fix errors automatically.

For example, you could provide the URL "https://news.ycombinator.com/" and the prompt "Extract all story titles, URLs, and point counts" and receive a complete, ready-to-use configuration file.

What models are supported for AI Configuration Generation?

The API supports a wide range of models through OpenRouter, including:

  • OpenAI models: GPT-4, GPT-3.5-turbo
  • Anthropic models: Claude-3-7-Sonnet, Claude-3-5-Sonnet
  • Google models: Gemini 2.5 Pro
  • DeepSeek models: DeepSeek Reasoner, DeepSeek Chat

You can specify your preferred model in the request options, and the system will automatically route to the appropriate provider.

How does the Navigation Engine work?

The Navigation Engine executes multi-step browser interactions defined in a configuration file. It can handle complex tasks like logging in, clicking buttons, filling forms, scrolling, handling popups, switching tabs/frames, and executing custom scripts. It uses Playwright for browser automation and supports various step types detailed in the documentation.

This powerful engine allows you to automate virtually any web interaction, making it possible to scrape even the most complex websites with dynamic content.

What kind of proxies does the API support?

The API features a robust Proxy Manager that can load proxies from various sources (currently JSON files, API sources planned). It supports:

  • Health checks and performance metrics tracking
  • Automatic rotation based on performance (latency, success rate)
  • Filtering by criteria like protocol, country, or anonymity level
  • Session-based proxy assignment for consistent identity
  • Detailed statistics and management via the Proxy API
Is this API suitable for beginners?

Yes! While the API offers advanced capabilities for experienced developers, the AI Configuration Generation feature makes it accessible to beginners. You can simply describe what data you want in natural language, and the API will generate the necessary configuration for you.

Additionally, we provide comprehensive documentation, ready-to-use templates, and step-by-step guides to help you get started quickly.

How can I deploy this on my own server?

The Advanced Web Scraper API is designed to be easy to deploy on your own infrastructure. You can:

  • Clone the repository and run it on your server
  • Use Docker for containerized deployment
  • Deploy to cloud services like AWS, Google Cloud, or Azure

Detailed deployment instructions are available in our documentation.
